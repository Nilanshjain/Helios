# Helios — Training and Operational Scripts

Scripts for generating labeled telemetry, training the production
anomaly-detection model, evaluating against public benchmarks, and
operating the live stack.

## What the model is trained on

The production Isolation Forest at `models/isolation_forest.pkl` is trained
on **Helios's own labeled telemetry**: events generated by
`generate_chaos_traffic.py`, posted through the live ingestion API,
persisted to TimescaleDB, then read back and windowed by
`train_production.py` via the same `FeatureExtractor` the live detection
consumer uses. Training and inference therefore share a single feature
pipeline — no second feature implementation, no train/inference
distribution gap.

NAB and SMD live under `scripts/datasets/` and are used by
`scripts/evaluate.py` for **benchmark evaluation** only — they don't feed
the production model.

## Quick start

```bash
# 1. Bring the stack up
make up   # or: docker compose up -d

# 2. Generate ~2 hours of labeled chaos telemetry (push to live ingestion)
python scripts/generate_chaos_traffic.py --duration-hours 2 --base-rps 100

# 3. Train on the events that just landed in TimescaleDB
python scripts/train_production.py \
    --timeline data/chaos/timeline_<UTC-timestamp>.json

# 4. Pick the model up live
docker compose restart detection detection-consumer
```

`make train-production` does steps 2 and 3 back-to-back with sensible
defaults.

## Scripts

### `generate_chaos_traffic.py`

Realistic per-service event generator with scripted chaos scenarios:
latency spikes, error storms, partial outages, traffic surges, cascading
timeouts, dependency failures. Posts JSON events to
`POST /api/v1/events` and writes a `data/chaos/timeline_*.json` describing
every injected scenario so the trainer can label windows.

Flags:

| Flag | Default | Notes |
|---|---|---|
| `--duration-hours` | `2.0` | Run length |
| `--base-rps` | `100` | Combined baseline RPS across all services |
| `--num-scenarios` | `8` | Chaos scenarios to inject |
| `--seed` | `42` | RNG seed for traffic + scenario placement |
| `--ingestion-url` | `http://localhost:8080` | Helios ingestion base URL |
| `--concurrency` | `20` | HTTP concurrency |
| `--timeline-out` | `data/chaos/timeline_<utc>.json` | Where to write the timeline |
| `--dry-run` | `false` | Print the plan and exit without posting |

### `train_production.py`

Reads events from TimescaleDB inside the timeline's time window, splits
them into non-overlapping 60s windows, labels each from the timeline,
trains `AnomalyDetector` via its existing `train()` method, sweeps the
threshold on a **temporal** validation split (last-20% time, not random
— anomalies cluster in time), saves the model to
`models/isolation_forest.pkl`.

Flags:

| Flag | Default | Notes |
|---|---|---|
| `--timeline` | required | Path to a timeline JSON |
| `--window-size-seconds` | `60` | Matches the live consumer |
| `--window-stride-seconds` | `60` | Non-overlapping by default |
| `--train-frac / --val-frac` | `0.6 / 0.2` | Test = remainder |
| `--contamination` | `0.05` | IsolationForest contamination |
| `--n-estimators` | `100` | Forest size |
| `--seed` | `42` | RNG seed (incl. SHAP background sampling) |
| `--output` | `models/isolation_forest.pkl` | Production model path |
| `--no-mlflow` | `false` | Skip MLflow logging |
| `--no-roundtrip-check` | `false` | Skip post-save load + predict sanity check |
| `--save-features` | `false` | Also dump `models/training_data.csv` for drift_check |

The script writes/refreshes:

- `models/isolation_forest.pkl` — production model (model + scaler + threshold + feature_names + SHAP background)
- `models/model_config.json` — training metadata + chosen threshold + per-split metrics (read by `detection-consumer` for the `helios_model_prediction_age_days` Prometheus gauge)
- `models/training_metrics.json` — same metrics for CI / drift_check consumers
- MLflow run under experiment `helios-production-train`

### `evaluate.py`

Benchmark the same `windows_to_features.py` pipeline + IsolationForest
hyperparameters against NAB and SMD public datasets. Used to characterize
the **modeling pipeline** on standard benchmarks, not to produce the
deployed model. Writes `models/evaluation/{results.json,model_card.md,*.png}`.

```bash
python scripts/evaluate.py --dataset both --contaminations 0.03 0.05 0.10
```

NAB and SMD are auto-downloaded into `data/` by
`scripts/datasets/download.py` on first run.

### `drift_check.py`

Population-Stability-Index (PSI) drift detection. Compares a *current*
feature distribution against the *reference* training distribution
(`models/training_data.csv` — produced when `train_production.py` is run
with `--save-features`). If no current data is provided, `--simulate
minor|moderate|severe` perturbs the reference for demo/smoke purposes.

```bash
python scripts/drift_check.py --simulate moderate
```

Exit code is `0` if max PSI ≤ 0.25, `1` otherwise.

### `continuous_traffic.py`

Low-volume continuous traffic (~10 events/sec) with periodic error
bursts. Useful for demos — easier on the machine than
`generate_chaos_traffic.py`, but it does NOT produce labeled telemetry
suitable for training.

### `load_test.py`

Stress test the ingestion API at configurable RPS. Used to measure
ingestion throughput / latency; does not produce labeled data.

### `dry_run.py`

End-to-end smoke check: verifies model file is present, model loads,
inference returns sensible features. Useful for CI.

### `demo.py`, `generate_demo_data.py`, `populate_demo_data.sql`, `simulate_indian_scenarios.py`, `live_anomaly_generator.py`, `generate_sample_report.py`, `diagnose_reporting.py`

Demo and reporting helpers — see their `--help` output.

### `generate_events.go`

Go-based event generator. Equivalent in spirit to the Python event
generators but compiles to a single binary.

## Directory layout

```
scripts/
├── datasets/                # NAB + SMD loaders + windows_to_features pipeline
├── continuous_traffic.py
├── demo.py
├── diagnose_reporting.py
├── drift_check.py
├── dry_run.py
├── evaluate.py              # benchmark eval (NAB + SMD)
├── generate_chaos_traffic.py  # labeled production-training data generator
├── generate_demo_data.py
├── generate_events.go
├── generate_sample_report.py
├── live_anomaly_generator.py
├── load_test.py
├── populate_demo_data.sql
├── README.md
├── setup.sh
├── simulate_indian_scenarios.py
└── train_production.py      # production trainer
```

`models/` (sibling of `scripts/`) is the canonical home for the trained
model, training metrics, drift artifacts, and the evaluation
sub-directory.

`data/` holds NAB/SMD datasets (auto-downloaded) and `data/chaos/`
(timeline JSONs from `generate_chaos_traffic.py`).
